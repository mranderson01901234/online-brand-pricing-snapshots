# 12-Hour Distribution Plan

Checklist for coordinated release across channels.

---

## Hour 0-1: GitHub Prep

- [ ] Set repository description: "Weekly pricing snapshots from 500+ online brand storefronts. Structured. Versioned. MIT licensed."
- [ ] Add topics: `datasets`, `ecommerce`, `retail-analytics`, `pricing`, `price-tracking`, `market-research`, `data-analysis`
- [ ] **Avoid topics:** `scraping`, `crawler`, `shopify`
- [ ] Verify README renders correctly
- [ ] Verify all doc links work
- [ ] Star your own repo (establishes initial activity)

## Hour 1-2: Reddit Post

- [ ] Post to r/datasets using copy from `reddit_post.md`
- [ ] Do not edit after posting
- [ ] Note the post URL for tracking

## Hour 2-3: Twitter Thread

- [ ] Post thread using copy from `twitter_thread.md`
- [ ] Post all 5 tweets in sequence
- [ ] Pin thread to profile
- [ ] Do not reply to anyone for 1 hour

## Hour 3-4: First GitHub Issue

- [ ] Submit issue to awesomedata/awesome-public-datasets
- [ ] Use copy from `github_issue_awesome-datasets.md`
- [ ] Check their contribution guidelines first (may need PR instead)
- [ ] Note the issue/PR URL

## Hour 4-5: Second GitHub Issue

- [ ] Submit issue to academic/awesome-datascience
- [ ] Use copy from `github_issue_awesome-datascience.md`
- [ ] Check their contribution guidelines first
- [ ] Note the issue/PR URL

## Hour 5-6: Third GitHub Issue (Optional)

- [ ] Submit issue to public-apis/public-apis
- [ ] Use copy from `github_issue_public-apis.md`
- [ ] This may be declined (dataset vs API scope)
- [ ] Only submit if time permits

## Hour 6-8: Light Engagement

- [ ] Check Reddit post for questions
- [ ] Reply factually to legitimate questions
- [ ] Do not argue with critics
- [ ] Check Twitter for replies
- [ ] Reply factually, briefly

## Hour 8-10: Monitor

- [ ] Check GitHub for stars, forks, issues
- [ ] Check if awesome-* issues/PRs were merged or commented
- [ ] Note any feedback for iteration

## Hour 10-12: Document Results

- [ ] Record final stats: Reddit upvotes, Twitter impressions, GitHub stars
- [ ] Note which channels performed best
- [ ] Document any questions that came up frequently
- [ ] Plan follow-up content if traction is good

## Response Templates

**"When is the data coming?"**
> First snapshot release is in progress. The repo and schema are published so people can evaluate the structure before data lands.

**"How is this collected?"**
> From publicly accessible storefronts. Full methodology is documented: https://getblueprint.io/methodology

**"Can you add [brand]?"**
> Coverage decisions are based on data quality and feasibility. We don't take requests, but scope is documented in each release.

**"Is this legal?"**
> We publish aggregated snapshots of publicly visible pricing. MIT licensed. Downstream use is the user's responsibility.
